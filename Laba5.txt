1)Кроки інсталяції та налаштування Tailwind CSS (v4):
+Встановлено необхідні пакети через термінал: npm install -D tailwindcss @tailwindcss/vite.
+Налаштовано збирач Vite: у файл vite.config.ts додано плагін tailwindcss().
+Створено головний CSS-файл (src/assets/css/index.css), у якому підключено фреймворк одним рядком: @import "tailwindcss";.
+Імпортовано цей CSS-файл у головний файл додатку main.ts, щоб стилі застосувалися до всього проєкту.

2)Реалізація умовних стилів у UserCard.vue.
Для зміни вигляду картки залежно від пропсу isActive було використано динамічне зв'язування класів Vue (:class).
Замість того щоб перемикати один CSS-клас (наприклад, .active), ми використали тернарний оператор для підстановки конкретних класів Tailwind.
Приклад коду::class="isActive ? 'border-green-500 bg-green-50' : 'border-gray-300 bg-white'"
Це дозволяє динамічно змінювати колір рамки та фону без написання кастомного CSS.

3)Переваги та недоліки utility-first підходу (Tailwind) порівняно з <style scoped>:
Переваги:
  +Швидкість розробки: не потрібно перемикатися між файлами та вигадувати назви для класів (як-от .card-wrapper, .text-box).
  +Уніфікація дизайну: використовуються стандартні відступи та кольори з палітри, що робить дизайн цілісним.
  +Менший розмір CSS: не створюються дубльовані стилі для кожного компонента.
Недоліки:
  +"Забруднення" HTML-коду: рядки з класами стають дуже довгими, що може ускладнювати читання структури шаблону.
  +Потрібен час на вивчення назв класів
